\Blazar\Component\WebService

\Blazar\Component\WebServiceWebService

Classe para disponibilizar APIs em rede atraves de requisições GET e POST.

As APIs disponíveis devem estar em manifest>map

Exemplo de requisições:

  • common: .../api_map_name/?method=(method_name)&(param1)=(value)&(param2)=(value)&...
  • multi_request: .../?multi_request&params={(request_key1):{method:(api_map_name/method_name),(param1):(value),(param2)=(value)}, (request_key2):{method:(api_map_name/method_name),(param1):(value),(param2)=(value)}, ...}
  • long_polling: .../?long_polling&params={(request_key1):{method:(api_map_name/method_name),(param1):(value),(param2)=(value)}, (request_key2):{method:(api_map_name/method_name),(param1):(value),(param2)=(value)}, ...}

    Os métodos das APIs irão receber os seguintes parâmetros:

    0 array [data] As informações da requisição.

    1 array|null [result_list] (Apenas para multi_request e long_polling) A lista de resultados gerados pelas

    requisições anteriores

    2 object|null [instance] (Apenas para webservice) A instancia da classe webservice.

__construct()

__construct(  webservice_controller = false,   method_param = null,   inherited_enable = false) :

WebService constructor.

Define automaticamente qual é o tipo de requisição dependo dos parametros informados.

Esta classe deve ser chamada apenas em classes listadas no map do manifest.
Para utilizar com multiplas apis, a classe controladora deve ser a ultima a ser chamada no map para que a API consiga identificar os descendentes dela.

Parameters

webservice_controller

A classe é um Webservice para controlar APIs cadastradas no seu "sub" no manifest.
Se true irá trabalhar com as classes descendentes da herdeira informadas no map->sub.
Se false apenas os métodos da classe herdeira serão validos.

method_param

Ação padrão para a chamada dos métodos das requisições
A classe API deve possuir um método com o nome enviado pelo parametro

Ex: .../user/?method=show_name
